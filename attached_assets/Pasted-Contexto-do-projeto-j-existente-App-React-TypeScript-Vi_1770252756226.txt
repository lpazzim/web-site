Contexto do projeto (já existente)
- App React + TypeScript (Vite) + Tailwind/shadcn.
- Rotas via react-router-dom em `src/App.tsx`.
- Home é `src/pages/Index.tsx` (visual editorial, classes como `container-wide`, `text-label`, etc. em `src/index.css`).
- Existe QueryClientProvider no App (tanstack/react-query disponível), framer-motion disponível.

Objetivo
1) Criar uma página de listagem de posts do blog: `/blog`
2) Criar uma página de leitura do post: `/blog/:slug` renderizando o conteúdo vindo do Notion (via API oficial).
3) Adicionar na Home (`src/pages/Index.tsx`) uma seção “Latest Posts” com os 3 últimos posts publicados, com link para `/blog` e para cada post.
4) Respeitar o campo `Published` (checkbox) do Notion: somente posts com Published=true aparecem na Home e na listagem; acesso direto em `/blog/:slug` de um post não publicado deve retornar 404 (ou uma tela equivalente).

Fonte de dados (Notion)
- Usar a integração do Notion JÁ cadastrada no ambiente do Replit (NUNCA expor o token no bundle do frontend).
- Banco de dados do Notion tem as propriedades (como no print):
  - Title (title)
  - Date (date)
  - Published (checkbox)
  - Slug (rich_text)
  - Description (rich_text)
  - Canonical (url) [opcional]
  - Image (url) [opcional]
  - Author (people) [opcional]
- Ordenação: por Date desc (mais recente primeiro). Em caso de ausência de Date, cair para `last_edited_time`.

Arquitetura recomendada (para não vazar segredo)
- Implementar um pequeno backend Node no mesmo Replit para chamar a Notion API (server-side) e expor endpoints REST para o frontend.
- Frontend consome apenas `/api/...`.
- Se já existir uma integração/infra de API no Replit deste projeto, reutilizar; caso não exista, criar sem quebrar `npm run dev` e `npm run build`.

Endpoints (mínimo)
1) GET `/api/blog/posts?limit=3`
   - Retorna array de posts publicados (Published=true), ordenados por Date desc.
   - Campos: id, title, slug, date, description, image, authorName (ou null), canonical (ou null)
2) GET `/api/blog/posts/:slug`
   - Busca o post pelo Slug no database.
   - Se não existir ou Published=false: retornar 404.
   - Retornar metadata + conteúdo do Notion page:
     - Preferência: retornar `contentMarkdown` (string) gerado a partir dos blocks do Notion.
     - Alternativa: retornar uma estrutura de blocks e renderizar no client (mas manter renderização completa e legível).

Implementação Notion (server-side)
- Usar `@notionhq/client` com token da integração (ler de Secrets/env do Replit).
- Descobrir/usar o `NOTION_DATABASE_ID` também via Secrets/env.
- Query do database:
  - Filter: Published == true (para listagens).
  - Sorts: Date descending (fallback last_edited_time).
- Para obter o conteúdo do post:
  - Encontrar page pelo Slug (filter no database onde Slug == param).
  - Listar blocks com paginação (`blocks.children.list`) e montar conteúdo.
  - Converter blocks para Markdown com uma lib (ex.: `notion-to-md`) e devolver markdown.
- Adicionar caching simples in-memory (ex.: 60s) para reduzir chamadas ao Notion.

Frontend (UI/rotas)
- Criar `src/pages/Blog.tsx` (listagem):
  - Usa Layout existente (`<Layout ...>`), mantém estética editorial (container-wide, tipografia, bordas, uppercase labels).
  - Renderiza cards/linhas com imagem (se houver), Title, Date, Description, Author.
  - Link para `/blog/:slug`.
- Criar `src/pages/BlogPost.tsx`:
  - Header do post: Title, Date, Author, imagem (se houver).
  - Corpo: render do markdown (ex.: `react-markdown` + `remark-gfm`). Garantir estilos básicos de leitura (largura confortável, espaçamento, headings, listas, quotes, code).
- Atualizar rotas em `src/App.tsx`:
  - Adicionar `<Route path="/blog" element={<Blog />} />`
  - Adicionar `<Route path="/blog/:slug" element={<BlogPost />} />`
- Home (`src/pages/Index.tsx`):
  - Inserir uma seção “Latest Posts” com os 3 últimos posts publicados.
  - Visual coerente com seções existentes (pode usar framer-motion/whileInView e `TextReveal`).
  - Ter link “View all” -> `/blog`.
  - Cada item: título + data + descrição curta + imagem opcional.

Regras de exibição (Published)
- Qualquer lista (Home e /blog) deve filtrar Published=true.
- /blog/:slug deve validar Published=true; caso false, mostrar NotFound (pode reutilizar `src/pages/NotFound.tsx` ou um 404 mais editorial).

Qualidade / critérios de aceite
- `npm run dev` funciona (frontend + api).
- `npm run build` funciona.
- Home mostra exatamente 3 posts mais recentes publicados.
- /blog lista somente publicados e em ordem correta.
- /blog/:slug renderiza conteúdo do Notion de forma legível e completa (pelo menos: headings, parágrafos, listas, links, imagens, code).
- Nenhum token do Notion fica exposto no client (sem `VITE_...` com segredo).
- Tratar loading/erro com UI simples e consistente.

Notas
- Se o nome das env vars já existir no Replit, reutilize; caso não exista, padronize como:
  - NOTION_TOKEN
  - NOTION_DATABASE_ID
- Não commitar/printar valores de Secrets.
