Objetivo
1) Criar pagina de listagem de posts do blog: `/blog`
2) Criar pagina de leitura do post: `/blog/:slug` renderizando o conteudo vindo do Notion
3) Adicionar na Home (`src/pages/Index.tsx`) uma secao com os 3 ultimos posts do blog (somente publicados)
4) Respeitar o campo `Published` (checkbox) do Notion para decidir se o artigo aparece no site:
   - Home e `/blog`: somente posts com Published=true
   - `/blog/:slug`: se o post nao existir ou Published=false, retornar 404 (pode reutilizar `src/pages/NotFound.tsx` ou uma tela 404 consistente)

Fonte de dados (Notion) — propriedades do database (como no print)
- Title (title)
- Date (date)
- Published (checkbox)
- Slug (rich_text)
- Description (rich_text)
- Canonical (url) opcional
- Image (url) opcional
- Author (people) opcional
Regras:
- Ordenar por Date desc (mais recente primeiro). Se Date estiver vazio, usar `last_edited_time` como fallback.
- Slug e usado na rota `/blog/:slug` e deve ser unico.

- Esse adapter deve prover 2 funcoes:
  1) `listPosts({ limit? })` -> retorna posts publicados (Published=true) ordenados
  2) `getPostBySlug(slug)` -> retorna post + conteudo; se Published=false, tratar como nao encontrado
- Adicionar um cache simples no client (react-query recomendado) para evitar re-fetch excessivo.

- Garantir layout de leitura:

  - Se houver Image do post, usar como hero/cover no topo.
